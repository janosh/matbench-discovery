> Disclaimer: Take the results on this page with a grain of salt. We checked most data but not as carefully yet as the discovery task.

The PBE/r2SCAN DFT reference curves shown below are documented in the [DFT reference provenance note](https://github.com/janosh/matbench-discovery/blob/main/site/src/lib/diatomics-dft.readme.md), including the VASP setup, spin-ladder generation, warm-start strategy, postprocessing, and quality checks. Expert advice on how the DFT reference curves might be improved would be appreciated!

Metrics in the table below aggregate over the same element set for all models: the 87 [Materials Project](https://materialsproject.org) elements between H and U. Po, At, Rn, Fr, Ra are excluded since they're absent from MP and hence unsupported by models trained on MP-derived data (MPtrj, OMat24, sAlex) &mdash; only a minority of models predicts them. Self-consistency metrics (smoothness, flips, jumps) average over all 87 elements. Reference-relative PBE errors skip 8 further lanthanides (Pr, Pm, Sm, Tb, Dy, Ho, Er, Tm) whose reference curves fail a smoothness quality gate (energy jump &ge; 1.5 eV and &ge; 3 energy-difference sign flips inside the scored window &mdash; f-electron SCF convergence issues make these references too jumpy to score models against). Curves excluded from metrics are still shown in the plots below. Models missing curves within the scored set (marked \*) are scored on their remaining elements pending backfill runs.

With thanks to [MLIP Arena](https://huggingface.co/spaces/atomind/mlip-arena#homonuclear-diatomics) by [Yuan Chiang](https://scholar.google.com/citations?user=kZ2LfBoAAAAJ&hl=en), who first benchmarked MLIPs on self-consistency metrics of homonuclear diatomic curves and inspired this task.

We welcome backfill PRs from model authors for models still missing diatomic metrics; see the [diatomics runner](https://github.com/janosh/matbench-discovery/blob/main/models/run_diatomics.py), [calculator registry](https://github.com/janosh/matbench-discovery/blob/main/matbench_discovery/calculators.py), and [model enum](https://github.com/janosh/matbench-discovery/blob/main/matbench_discovery/enums.py).
